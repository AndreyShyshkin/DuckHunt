Реалізація класичної гри **Duck Hunt** мовою Python (Pygame) із впровадженням повного циклу CI/CD (Plan, Code, Build, Test).

## Встановлення та Запуск

Проєкт використовує сучасну структуру `src-layout`. Для коректної роботи необхідно встановити проєкт у режим розробки (Editable mode).

### 1. Клонування та підготовка

```Bash
git clone https://github.com/AndreyShyshkin/DuckHunt/
cd DuckHunt
```

### 2. Встановлення залежностей

Ми встановлюємо залежності та сам проєкт як локальний пакет:

```Bash
pip install -r requirements.txt
pip install -e .
```
Важливо: Команда pip install -e . є обов'язковою, оскільки вона налаштовує імпорти з папки src.

### 3. Запуск гри

```Bash
python run.py
```

### Аргументи командного рядка
Ви можете налаштувати гру при запуску за допомогою наступних аргументів:

* `--width`: Встановити ширину екрану (за замовчуванням: 800)
* `--height`: Встановити висоту екрану (за замовчуванням: 500)
* `--volume`: Встановити гучність звуків (від 0.0 до 1.0, за замовчуванням: 1.0)
* `--difficulty`: Встановити рівень складності (швидкість птахів) (1: Нормально, 2: Швидко, 3: Важко, за замовчуванням: 1)
* `--fps`: Встановити кількість кадрів на секунду (за замовчуванням: 60)

Приклад запуску з користувацькими налаштуваннями:
```bash
python run.py --width 1024 --height 768 --volume 0.5 --difficulty 2 --fps 120
```

## Тестування та перевірка коду
У проєкті налаштовано автоматичний Pipeline, який запускається при кожному коміті. Ви можете запустити перевірки локально.

### Запуск Unit-тестів
Ми використовуємо pytest для тестування. Щоб запустити всі тести:

```
pytest
```

### Перевірка стилю коду (Linter)
Ми використовуємо flake8 для перевірки відповідності PEP8.

```
flake8 .
```
